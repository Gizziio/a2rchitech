## What Agentation IS

**Agentation** (agentation.com) is a **visual annotation overlay** for UI development:

- Runs in browser as dev-only overlay
- Lets you click DOM elements + text ranges to attach comments
- Produces copy/paste "instructions + selectors/context" for coding agents
- **NOT** an agent execution wrapper
- **NOT** a runtime component

---

## What Agentation IS NOT

- ❌ NOT an agent execution framework
- ❌ NOT a policy enforcement layer
- ❌ NOT a core A2R runtime component
- ❌ NOT something that belongs in `1-kernel/execution/`

---

## Correct Integration Approach

### 1. Install as Dev Dependency

```bash
# In UI package (e.g., 6-ui/a2r-platform or 7-apps/shell-ui)
npm install agentation -D
```

### 2. Mount Only in Development

```typescript
// 6-ui/a2r-platform/src/main.tsx (or equivalent entry point)
import { Agentation } from 'agentation';

if (process.env.NODE_ENV !== 'production') {
  // Mount Agentation overlay
  Agentation.mount({
    apiKey: process.env.AGENTATION_API_KEY,
    projectId: 'a2r-platform',
  });
}
```

### 3. Enable in Storybook

```typescript
// .storybook/preview.tsx (or .storybook/preview.js)
import { Agentation } from 'agentation';

if (process.env.NODE_ENV !== 'production') {
  export const decorators = [
    (Story) => (
      <>
        <Agentation />
        <Story />
      </>
    ),
  ];
}
```

### 4. A2R-Specific Adapter

Create a wrapper that appends A2R execution context to Agentation output:

```typescript
// 6-ui/a2r-platform/src/dev/agentation-adapter.ts

interface AgentationOutput {
  notes: string;
  selectors: string[];
  context: string;
}

interface A2RExecutionHeader {
  uiSurface: 'apps/shell' | 'packages/ui';
  storyId?: string;
  route?: string;
  renderer: 'HUMAN' | 'AGENT';
  viewport: { width: number; height: number };
  acceptance: string;
}

export function wrapForA2R(
  output: AgentationOutput,
  header: A2RExecutionHeader
): string {
  return `
# A2R UI Annotation

## Execution Context
- UI Surface: ${header.uiSurface}
- Story: ${header.storyId || 'N/A'}
- Route: ${header.route || 'N/A'}
- Renderer: ${header.renderer}
- Viewport: ${header.viewport.width}x${header.viewport.height}

## Acceptance Criteria
${header.acceptance}

## Agentation Notes
${output.notes}

## Selectors
${output.selectors.join('\n')}

## Context
${output.context}
`.trim();
}
```

## Workflow

1. **Designer/Developer** opens Storybook story
2. **Clicks Agentation toggle** (dev-only)
3. **Selects UI element** to annotate
4. **Adds notes** describing desired change
5. **Copies structured output**
6. **Pastes into A2R agent** (e.g., UI_IMPLEMENTER agent)
7. **Agent executes** with full context (selectors + notes + A2R header)
8. **Storybook rebuilds** + interaction tests verify

## Benefits for A2R

| Benefit | Description |
|---------|-------------|
| **Precise targeting** | Click exact element, get selector |
| **Visual context** | Screenshot + DOM context included |
| **Structured output** | Copy/paste ready for agents |
| **Dev-only** | Zero production impact |
| **Storybook integration** | Works with isolated component states |

| Interference with normal interaction | Toggle/hotkey to enable/disable |
