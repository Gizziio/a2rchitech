## Overview

## Core API

### Policy Tiers

```rust
pub enum PolicyTier {
    Minimal,       // Public data, safe operations
    Standard,      // Internal data, common operations
    Elevated,      // Sensitive data
    HighAssurance, // Critical operations
    Critical,      // Highest risk operations
}
```

### Tier Requirements

| Tier | Approvals Required | Max Risk | Human Oversight | Max Duration |
|------|-------------------|----------|-----------------|--------------|
| Minimal | None | Low | No | 1 hour |
| Standard | 1 Operator | Moderate | No | 30 min |
| Elevated | 1 Senior Operator | High | Yes | 15 min |
| HighAssurance | 1 Senior + 1 Security | Critical | Yes | 5 min |
| Critical | 2 Senior + 1 Security + 1 Chief | Critical | Yes | 1 min |

### PolicyTierRegistry

```rust
let registry = PolicyTierRegistry::new();

// Create tier assignment
let assignment = registry
    .create_assignment(
        PolicyTier::Standard,
        TargetType::Tool,
        "tool-1",
        "admin",
        "Standard tool execution",
        risk_assessment,
    )
    .await?;

// Add approval
registry
    .add_approval(
        &assignment.assignment_id,
        Approval {
            approver_id: "operator-1".to_string(),
            role: "operator".to_string(),
            approved_at: Utc::now(),
            comments: "Approved".to_string(),
        },
    )
    .await?;

// Check if required approvals are met
let has_approvals = registry
    .has_required_approvals(&assignment.assignment_id)
    .await?;

// Perform gate check
let request = GateCheckRequest {
    request_id: "req-1".to_string(),
    target_type: TargetType::Tool,
    target_id: "tool-1".to_string(),
    operation: "execute".to_string(),
    context: OperationContext {
        actor_id: "user-1".to_string(),
        session_id: "session-1".to_string(),
        data_sensitivity: vec!["internal".to_string()],
        environment: "production".to_string(),
        metadata: HashMap::new(),
    },
    requested_tier: PolicyTier::Standard,
};

let result = registry.gate_check(request).await?;
if result.allowed {
    // Proceed with operation
} else {
    // Handle violations and escalations
}

// Elevate tier (if needed)
let elevated = registry
    .elevate_tier(&assignment.assignment_id, PolicyTier::HighAssurance, "admin", "Critical operation")
    .await?;

// Revoke assignment
registry.revoke_assignment(&assignment.assignment_id, "admin").await?;

// Query assignments
let all = registry.get_all_assignments().await;
let by_tier = registry.get_assignments_by_tier(&PolicyTier::Standard).await;
let audit_log = registry.get_audit_log().await;
```

## SYSTEM_LAW Compliance

### LAW-TOOL-002 (HARD) — Capability-Based Policy Binding

**Requirement:** Policy enforcement must be bound to tool capability class.

**Implementation:**

| Capability | PreToolUse Gate | Approval Required | Receipt Required |
|------------|-----------------|-------------------|------------------|
| Read | Schema validation | No (Minimal tier) | Yes |
| Write | Schema + path validation | Risk tier ≥3 (Elevated) | Yes |
| Execute | Full policy check | Risk tier ≥2 (Standard) | Yes |
| Destructive | Security review | Always (human for tier 4) | Yes |
| External | Allowlist check | Risk tier ≥2 (Standard) | Yes |

### LAW-AUT-006 (HARD) — Risk Tier Classification

**Requirement:** All operations must be classified by risk tier.

**Implementation:**
- `RiskAssessment` with canonical risk levels
- `TierRequirements::for_tier()` provides tier-specific constraints
- Gate check validates risk level against tier limits

## Usage Example

```rust
use a2r_policy_tier_gating::{
    PolicyTierRegistry, PolicyTier, TargetType, RiskAssessment,
    RiskLevel, GateCheckRequest, OperationContext, Approval,
};

// Initialize registry
let registry = PolicyTierRegistry::new();

// Create risk assessment
let risk_assessment = RiskAssessment {
    level: RiskLevel::Moderate,
    factors: vec!["writes to filesystem".to_string()],
    mitigations: vec!["limited to /tmp".to_string()],
    residual_risk: RiskLevel::Low,
};

// Create tier assignment for a tool
let assignment = registry
    .create_assignment(
        PolicyTier::Standard,
        TargetType::Tool,
        "write_file",
        "security_admin",
        "Standard file write operations",
        risk_assessment,
    )
    .await?;

// Add required approval
registry
    .add_approval(
        &assignment.assignment_id,
        Approval {
            approver_id: "operator-1".to_string(),
            role: "operator".to_string(),
            approved_at: Utc::now(),
            comments: "Approved for production use".to_string(),
        },
    )
    .await?;

// Before tool execution, perform gate check
let request = GateCheckRequest {
    request_id: "gate-1".to_string(),
    target_type: TargetType::Tool,
    target_id: "write_file".to_string(),
    operation: "write".to_string(),
    context: OperationContext {
        actor_id: "agent-1".to_string(),
        session_id: "session-123".to_string(),
        data_sensitivity: vec!["internal".to_string()],
        environment: "production".to_string(),
        metadata: HashMap::new(),
    },
    requested_tier: PolicyTier::Standard,
};

let result = registry.gate_check(request).await?;

if result.allowed {
    println!("Gate check passed, proceeding with execution");
} else {
    println!("Gate check failed:");
    for violation in &result.violations {
        println!("  - {}: {}", violation.violation_type, violation.message);
    }
    
    // Handle escalations
    for escalation in &result.required_escalations {
        println!("  Escalation required to {:?}: {}", escalation.to_tier, escalation.reason);
    }
}

// Query audit log
let audit_log = registry.get_audit_log().await;
for entry in &audit_log {
    println!("{:?}: {} by {}", entry.action, entry.details, entry.actor_id);
}
```
