## UI Stories Contract (`ui/STORIES.md`)

### Required Story States

Every component MUST have stories for:

1. **Default State** - Normal component rendering
2. **Loading State** - Loading indicator
3. **Error State** - Error handling
4. **Empty State** - Empty data handling
5. **Overflow State** - Large data sets
6. **Accessibility State** - A11y compliance

### Hard Fail Conditions

CI will **REJECT** any PR where:

- Stories missing for new component
- Required states not covered
- Storybook build fails
- Interaction tests fail
- Visual regressions unapproved
- Accessibility violations

## UI Evidence Lane Script

### Usage

```bash
# Run complete UI evidence lane
npm run evidence:ui [component_path] [wih_id] [dag_node_id]

# Example
npm run evidence:ui src/components/Button wih_123 node_456
```

### Pipeline Steps

1. **Build Storybook** - Compile stories
2. **Interaction Tests** - Test user interactions
3. **Visual Regression** - Chromatic comparison
4. **Accessibility Tests** - A11y compliance
5. **Count Stories** - Coverage metrics
6. **Emit Receipt** - Evidence artifact

### Evidence Receipt

```json
{
  "type": "ui:evidence",
  "timestamp": "2026-02-21T12:00:00Z",
```

## Harness Integration

### Integration with Harness Engineering

The UI Evidence Lane integrates with the Harness validation system:

```rust
// Pseudo-code for harness integration
async fn validate_ui_changes(wih_id: &str, dag_node_id: &str) -> Result<(), HarnessError> {
    // Run UI evidence lane
    let evidence_receipt = run_ui_evidence_lane(wih_id, dag_node_id).await?;
    
    // Validate all checks passed
    if evidence_receipt.evidence.storybook_build != "success" {
        return Err(HarnessError::ValidationFailed("Storybook build failed"));
    }
    
    if evidence_receipt.evidence.interaction_tests != "passed" {
        return Err(HarnessError::ValidationFailed("Interaction tests failed"));
    }
    
    if evidence_receipt.evidence.accessibility != "passed" {
        return Err(HarnessError::ValidationFailed("Accessibility tests failed"));
    }
    
    // Emit receipt to WIH
    emit_receipt_to_wih(wih_id, evidence_receipt).await?;
    
    Ok(())
}
```
