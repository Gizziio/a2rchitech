## Overview

The ShellUI BrowserView crate provides a native browser view component for ShellUI with agentic browsing capabilities. It supports deterministic, policy-gated browser automation integrated with A2R DAG/WIH workflows.

**Key Features:**
- Browser navigation with history management
- Session management with tabs and cookies
- Screenshot/capture functionality
- A2UI action handlers for agent-powered browsing
- Renderer separation (HUMAN vs AGENT)
- Integration with receipts schema and policy tier gating

## Usage Example

```rust
use a2r_shellui_browserview::{
    BrowserViewEngine, BrowserViewConfig, BrowserAction,
    RendererType, A2UIAction, A2UIActionHandler,
};
use std::sync::Arc;

// Initialize engine
let config = BrowserViewConfig {
    initial_url: Some("about:blank".to_string()),
    enable_agent_mode: true,
    ..Default::default()
};

let engine = BrowserViewEngine::new(config);

// Navigate to page (human renderer)
engine.execute_action(BrowserAction::Navigate {
    url: "https://example.com".to_string(),
    renderer: RendererType::Human,
}).await?;

// Agent-powered extraction
let engine_arc = Arc::new(engine);
let handler = A2UIActionHandler::new(engine_arc.clone());

let action = A2UIAction::Extract {
    selection: "Product information".to_string(),
    schema: serde_json::json!({
        "type": "object",
        "properties": {
            "name": {"type": "string"},
            "price": {"type": "number"}
        }
    }),
};

let result = handler.execute(action).await?;
println!("Extracted: {:?}", result.result);

// Take screenshot
engine.execute_action(BrowserAction::Screenshot {
    full_page: true,
}).await?;

// Check state
let state = engine.get_state().await;
println!("Current URL: {}", state.current_url);
println!("History entries: {}", state.history.len());
```
