## Overview

## Core API

### Receipt Structure

```rust
pub struct Receipt {
    pub receipt_id: String,              // Unique receipt ID
    pub created_at: DateTime<Utc>,       // Creation timestamp
    pub run_id: String,                  // Execution instance
    pub workflow_id: String,             // Workflow identifier
    pub node_id: String,                 // Node in workflow
    pub wih_id: String,                  // Work Item Header ID
    pub tool_id: String,                 // Tool identifier
    pub kind: String,                    // Receipt kind (tool_call_pre, tool_call_post, etc.)
    pub tool_def_hash: String,           // Hash of tool definition
    pub policy_decision_hash: String,    // Hash of policy decision
    pub pretool_event_hash: String,      // Hash of pre-tool event
    pub input_hashes: Vec<String>,       // Input hashes
    pub output_hashes: Vec<String>,      // Output hashes
    pub artifact_manifest: Vec<ArtifactEntry>,  // Artifact manifest
    pub write_scope_proof: WriteScopeProof,     // Write scope proof
    pub execution: ExecutionDetails,            // Execution details
    pub idempotency_key: String,                // Idempotency key
    pub retry_count: u32,                       // Retry count
    pub trace_id: String,                       // Distributed trace ID
    pub correlation_id: String,                 // Cross-event correlation
    pub environment_hash: String,               // Environment hash
    pub metadata: HashMap<String, Value>,       // Additional metadata
}
```

## SYSTEM_LAW Compliance

### LAW-AUT-004 (HARD) — Evidence/Receipts Queryability Rule

**Requirement:** The system must support querying receipts to decide next actions deterministically.

**Implementation:**

```rust

// Ralph Loop Compliance:
// ✅ Query receipts before retrying a WIH (avoid duplication)
// ✅ Query receipts to determine if node is complete
// ✅ Use receipts as canonical truth (not local state)
```

### LAW-EVD-001 (HARD) — Evidence Immutability Law

**Requirement:** All evidence artifacts must be immutable and hash-verified.

**Implementation:**
- Receipts are written once and never modified
- All hashes are SHA-256 cryptographic digests
- Write scope proof validates declared vs actual paths
- Bundle aggregate hash verifies receipt collection integrity

## Usage Example

```rust
use a2r_receipts_schema::{
    ReceiptManager, ReceiptBuilder, ReceiptQuery,
    calculate_hash,
};

// Initialize manager
let manager = ReceiptManager::new("/var/a2r/receipts");

// Before tool execution
let pre_receipt = manager
    .create_receipt("run-123", "workflow-456")
    .node_id("node-789")
    .wih_id("wih-001")
    .tool_id("read_file")
    .kind("tool_call_pre")
    .correlation_id("corr-abc")
    .tool_def_hash(calculate_hash(tool_def))
    .build();

manager.store_receipt(&pre_receipt).await?;

// After tool execution
let post_receipt = manager
    .create_receipt("run-123", "workflow-456")
    .node_id("node-789")
    .wih_id("wih-001")
    .tool_id("read_file")
    .kind("tool_call_post")
    .correlation_id("corr-abc")
    .input_hashes(vec![calculate_hash(input)])
    .output_hashes(vec![calculate_hash(output)])
    .execution(0, calculate_hash(stderr), calculate_hash(stdout), 150)
    .add_artifact("/tmp/output.txt", calculate_hash(file), 2048, "text/plain")
    .write_scope(
        vec!["/tmp/output.txt".to_string()],
        vec!["/tmp/output.txt".to_string()]
    )
    .build();

manager.store_receipt(&post_receipt).await?;

// Query receipts before retry (avoid duplication)
let query = ReceiptQuery::by_run("run-123")
    .with_pagination(100, 0);
let existing = manager.query_receipts(&query).await?;

if existing.iter().any(|r| r.kind == "tool_call_post" && r.tool_id == "read_file") {
    // Already executed, skip
}

// Validate receipt
let report = manager.validate_receipt(&post_receipt).await?;
assert!(report.valid);
```
