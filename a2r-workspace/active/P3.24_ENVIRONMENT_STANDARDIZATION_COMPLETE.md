## Core API

### EnvironmentManager

```rust
use a2r_environment_standardization::{
    EnvironmentManager, NormalizedEnvironmentSpec, EnvironmentBase,
    Toolchain, Workspace, CompatibilityTier,
};
use a2rchitech_system_law::SystemLawEngine;

// Create manager
let system_law = Arc::new(SystemLawEngine::new());
let manager = EnvironmentManager::new(system_law);

// Create environment spec
let spec = NormalizedEnvironmentSpec {
    api_version: "a2r.dev/v1".to_string(),
    name: "my-env".to_string(),
    base: EnvironmentBase::Native {
        image: "rust:1.76".to_string(),
    },
    toolchain: Toolchain {
        node: Some("20.11".to_string()),
        rust: Some("1.76".to_string()),
        ..Default::default()
    },
    workspace: Workspace {
        root: "/workspace".to_string(),
    },
    env: HashMap::new(),
    secrets: vec![],
    services: vec![],
    tasks: vec![],
    ports: None,
    checks: vec![],
    artifacts: None,
    resources: None,
    network: None,
    determinism: None,
    metadata: None,
};

// Create environment instance
let instance = manager
    .create_environment(
        spec,
        CompatibilityTier::EnterpriseBYOC,
        Some("run_001".to_string()),
        Some("sess_001".to_string()),
    )
    .await?;

// Run lifecycle
while instance.stage != LifecycleStage::Completed {
    manager.run_lifecycle(&instance.instance_id).await?;
}

// Get instance
let instance = manager.get_instance(&instance.instance_id).await;

// List instances by tier
let instances = manager.list_instances_by_tier(CompatibilityTier::LocalDev).await;
```

## Lifecycle Stages

```
Resolving → Provisioning → Bootstrap → Starting → Ready → Running → Collecting → Teardown → Completed
```

**Stage Descriptions:**
1. **Resolving** - Parse and validate NES
2. **Provisioning** - Create isolated environment
3. **Bootstrap** - Run tasks with triggers
4. **Starting** - Start services with health checks
5. **Ready** - Run readiness checks
6. **Running** - Execute main tasks
7. **Collecting** - Gather artifacts
8. **Teardown** - Destroy environment
9. **Completed** - Finished successfully

## Determinism Hashing

### Hash Components

| Hash | Components |
|------|------------|
| **envHash** | NES canonical JSON + base image digests |
| **policyHash** | Compatibility tier + policy doc |
| **inputsHash** | Toolchain versions + env vars |
| **codeHash** | Workspace commit SHA |

### Usage

```rust
// All hashes computed automatically
assert!(!instance.env_hash.is_empty());
assert!(!instance.policy_hash.is_empty());
assert!(!instance.inputs_hash.is_empty());
assert!(!instance.code_hash.is_empty());

// Same spec produces same hashes
assert_eq!(instance1.env_hash, instance2.env_hash);
```
