1️⃣ Explicit Directory & Artifact Map (Physical Reality)

Why this matters

You’ve defined everything-is-a-file, but builders will still ask:

“Which files exist where, and which layer owns them?”

Without this, people invent folders.

Add to MD-016

A canonical filesystem contract (even if partially virtual).

## Canonical Context Filesystem Map

/ctx/
  /journal/            # append-only event + artifact index
  /intent-graph/       # normalized intent nodes + edges
  /patterns/           # PatternSpec (draft → active → deprecated)
  /directives/         # compiled directive artifacts
  /flowir/             # canonical execution IR
  /runs/               # run manifests + outcomes
  /miniapps/           # structured ops surfaces (schemas only)
  /receipts/           # UTI + action receipts
  /summaries/          # derived, non-authoritative compressions
  /indexes/            # retrieval indexes (binary, int8, leann)

Rule to state explicitly

Files may be projected virtually, but logical paths are canonical.

This single section saves months later.

2️⃣ Retrieval Decision Table (Operational, Not Conceptual)

You already describe binary+int8 vs LEANN.
What’s missing is when each is used.

Add to MD-016

## Retrieval Routing Rules

| Query Type | Source | Retrieval Path |
|----------|--------|----------------|
| Recent run context | /ctx/runs | Binary + int8 |
| Active intent graph | /ctx/intent-graph | Binary + int8 |
| Pattern matching | /ctx/patterns | Binary + int8 |
| Long-term research | /ctx/journal | LEANN |
| Cold archive | /ctx/journal | LEANN |
| UI filtering | materialized views | No embeddings |

Hard rule

No agent chooses retrieval strategy.
Retrieval is policy, not intelligence.

This prevents “smart agents” from becoming nondeterministic.

4️⃣ Minimal Build Checklist (For Engineers & Agents)

This is the most underrated but highest leverage addition.

Add to MD-016

## Minimal Build Checklist (v1)

To claim MD-016 compliance, an implementation must:

- [ ] Store all state as addressable artifacts
- [ ] Never preload full context
- [ ] Enforce compilation before execution
- [ ] Separate spec, IR, and runtime
- [ ] Journal every execution
- [ ] Route retrieval by policy
- [ ] Project UI strictly from artifacts
- [ ] Make summaries non-authoritative

This becomes a compliance gate, not documentation.
