## Context Commands Examples

*Chapter: 4-advanced-copilot-agent-mode | Words: 530*

GitHub Copilot Chat’s context commands Git & Codebase Context • #changes → Shares your current Git changes (modified, staged, unstaged). Prompt Example : • “#changes Summarize my recent changes.” • “#changes Generate commit message for these changes.” • #codebase → Provides Copilot access to your entire project codebase. Prompt Example : • “#codebase Where is the login function defined?” • “#codebase Find all classes extending BaseController.” • #githubRepo → Gives repo-level context (branches, PRs, issues). Prompt Example : • “#githubRepo Show open pull requests.” • “#githubRepo Draft PR description for my latest branch.” Python Environment • #configurePythonEnvironment → Helps set up Python environment. Prompt Example : • “#configurePythonEnvironment Fix missing dependencies.” • “#configurePythonEnvironment Suggest Python version for TensorFlow.” • #getPythonEnvironmentInfo → Fetches environment details. Prompt Example : • “#getPythonEnvironmentInfo Which Python version am I using?” • “#getPythonEnvironmentInfo List installed libraries.” • #getPythonExecutableCommand → Finds Python command (python3, py, etc.). Prompt Example : • “#getPythonExecutableCommand Which command runs my Python here?” • #installPythonPackage → Installs missing Python packages. Prompt Example : • “#installPythonPackage Install requests and pandas.” Editing & Files • #editFiles → Allows Copilot to propose file edits. Prompt Example : • “#editFiles Refactor app.py to use logging instead of print.” • #extensions → Context about installed VS Code extensions. Prompt Example : • “#extensions Recommend extensions for Python testing.” • #fetch → Gets external resources (web pages, APIs). Prompt Example : • “#fetch https://jsonplaceholder.typicode.com/todos/1 and explain response.” • #findTestFiles → Locates test files in project. Prompt Example : • “#findTestFiles List all Jest test files.” • #new → Creates a new file. Prompt Example : • “#new Create a new file calculator.py with add/subtract functions.” Running & Debugging • #openSimpleBrowser → Opens a simple in-editor browser view. Prompt Example : • “#openSimpleBrowser Load my localhost server.” • #problems → Shows errors/warnings from VS Code Problems tab. Prompt Example : • “#problems Fix type errors in my TypeScript files.” • #runCommands → Executes terminal commands. Prompt Example : • “#runCommands Run npm install.” • #runNotebooks → Runs Jupyter notebooks. Prompt Example : • “#runNotebooks Execute all cells in analysis.ipynb.” • #runTasks → Runs VS Code tasks. Prompt Example : • “#runTasks Run build task.” • #runTests → Executes tests in your project. Prompt Example : • “#runTests Run all Jest tests.” Search & Debug • #search → Searches codebase for keywords. Prompt Example : • “#search Find useEffect hooks.” • #searchResults → Provides search results context. Prompt Example : • “#searchResults Explain how results match my query.” • #terminalLastCommand → Shares your last executed terminal command. Prompt Example : • “#terminalLastCommand Why did my last command fail?” • #terminalSelection → Shares selected terminal text. Prompt Example : • “#terminalSelection Explain this error message.” • #testFailure → Provides failing test details. Prompt Example : • “#testFailure Debug this failed test.” Productivity Helpers • #todos → Lists TODO comments in code. Prompt Example : • “#todos Show me all pending TODOs.” • #usages → Finds where a function/class is used. Prompt Example : • “#usages Show all usages of getUserData.” • #vscodeAPI → Accesses VS Code API for extensions. Prompt Example : • “#vscodeAPI Show how to create a new webview panel.”

## Copilot Internal Commands

*Chapter: 5-copilot-chat-commands-extensions | Words: 458*

Copilot Agent “skills” (internal commands) File & Workspace • apply_patch → Apply code changes (a “diff/patch”) to your current file(s). • create_directory → Make a new folder in your workspace. • create_file → Create a brand-new file with the name you specify. • create_new_workspace → Start a new empty VS Code workspace/ project. • edit_notebook_file → Open & modify an existing Jupyter Notebook file. • insert_edit_into_file → Insert new code/text into an existing file at the right spot. • list_dir → Show what’s inside a folder (like ls or dir). • read_file → Read the contents of a file so Copilot can use it. Notebooks • create_new_jupyter_notebook → Start a new .ipynb Jupyter Notebook. • copilot_getNotebookSummary → Summarize what’s inside a notebook (cells, code, text). • run_notebook_cell → Execute a notebook cell and show results. Search & Info • file_search → Search text inside files in your workspace. • test_search → Look for test files or test cases in your project. • grep_search → Do a plain text search (like using grep in terminal). • semantic_search → Smarter search based on meaning (not just exact words). • get_search_view_results → Get results from VS Code’s built-in search panel. • list_code_usages → Find all the places a function/variable is used in your code. • vscode_searchExtensions_internal → Search the VS Code marketplace for extensions. Project & Errors • get_changed_files → Show files changed in git since last commit. • get_errors → Collect compiler/linter errors from your code. • get_project_setup_info → Get info about project setup (language, framework, config). • test_failure → Inspect a test failure to understand why it broke. Web & GitHub • fetch_webpage → Pull in content from a web page (useful for docs/ examples). • open_simple_browser → Open a basic web browser inside VS Code. • github_repo → Interact with the GitHub repo (issues, PRs, commits, etc.). Terminal & Commands • get_terminal_last_command → See the last terminal command you ran. • get_terminal_selection → Get whatever text is currently highlighted in terminal. • get_terminal_output → Grab the full output from the terminal. • run_in_terminal → Run a command in your VS Code terminal. • create_and_run_task → Create a VS Code task (like build/test) and run it. • get_task_output → Fetch the results of a task that was run. • run_vscode_command → Run any VS Code command programmatically. Python Tools • configure_python_environment → Set up Python environment (interpreter, venv, etc.). • get_python_environment_details → Show details about your Python environment. • get_python_executable_details → Get path/version of the Python interpreter. • install_python_packages → Install packages (like pip install numpy). Extensions & To-Do • install_extension → Install a VS Code extension. • manage_todo_list → Keep a running list of tasks/TODOs inside Copilot. Testing • runTests → Run your test suite and show results.
