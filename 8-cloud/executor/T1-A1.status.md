## Architecture Document Summary

The existing `ARCHITECTURE.md` document includes:

### Components Defined

1. **Executor Service** (`src/executor.rs`)
   - Job submission and validation
   - Lifecycle coordination
   - Status tracking
   - State machine: PENDING → QUEUED → STARTING → RUNNING → COMPLETED/FAILED/CANCELLED → TERMINATED

2. **Runtime Manager** (`src/runtime.rs`)
   - Docker container operations via bollard
   - Image pulling and management
   - Container lifecycle (create, start, stop, remove)
   - Log streaming

3. **JobQueue Service** (`src/job_queue.rs`)
   - Priority queue management
   - Scheduling policy (priority-based, fair scheduling)
   - Job routing

4. **ResourceManager** (`src/resource_manager.rs`)
   - CPU, memory, disk tracking
   - Resource allocation and release
   - Utilization metrics
