The Machine Learning Workflow

Machine learning projects follow a systematic workflow. Understanding this process helps you approach ML problems methodically and avoid common pitfalls.

The 6-Step ML Workflow

1

Define the Problem

Clearly articulate what you want to predict or optimize. Is it a classification problem (categories) or regression (continuous values)? What business or research question are you trying to answer?

2

Collect Data

Gather relevant data from databases, APIs, web scraping, or manual collection. The quality and quantity of data often matters more than algorithm choice.

3

Prepare Data

Clean the data: handle missing values, remove duplicates, fix errors. Transform features: scale, encode categories, create new features. Split into training, validation, and test sets.

4

Choose and Train Model

Select an appropriate algorithm based on your problem type, data size, and requirements. Train the model on your training data—this is where the learning happens.

5

Evaluate Model

Test the trained model on unseen data (test set). Use appropriate metrics to measure performance. Analyze errors to understand model limitations.

6

Deploy and Monitor

Integrate the model into production systems. Monitor performance over time as data distributions change (concept drift). Retrain periodically with new data.

⚠️ Important: This workflow is iterative, not linear. You'll often loop back to earlier steps—discovering you need more data, trying different features, or selecting a different model based on evaluation results.
2.2

Data Collection and Preprocessing

Data is the foundation of machine learning. The phrase "garbage in, garbage out" applies perfectly—your model can only be as good as the data you train it on.

Data Quality Issues

❌ Missing Values

Data points with empty or null values. Solutions: imputation, deletion, or using algorithms that handle missing data.

❌ Outliers

Extreme values that deviate from the pattern. Can be errors or genuine anomalies—handle carefully.

❌ Duplicates

Identical or near-identical records. Can inflate performance metrics artificially if they appear in both train and test sets.

❌ Inconsistent Formats

Different representations of the same information (e.g., "USA", "US", "United States").

Feature Engineering

Feature engineering is the process of transforming raw data into features that better represent the underlying problem, improving model performance.

Common Transformations:

  • Scaling: Normalize or standardize numerical features to similar ranges
  • Encoding: Convert categorical variables to numerical (one-hot, label encoding)
  • Binning: Group continuous values into discrete categories
  • Feature Creation: Combine existing features (e.g., BMI from height and weight)
  • Dimensionality Reduction: Reduce number of features while preserving information

Train-Validation-Test Split

We split data into three sets to ensure our model generalizes to new data:

Training Set (~70%)

Used to train the model—adjust weights and learn patterns.

Validation Set (~15%)

Used to tune hyperparameters and compare models. Not used for training.

Test Set (~15%)

Used only once at the end to evaluate final model performance. Must remain untouched during development.

💡 Best Practice: Use stratified sampling for classification problems to ensure each split maintains the same class distribution as the original dataset.

Evaluation Metrics and Model Performance

How do you know if your model is good? Evaluation metrics quantify performance, but choosing the right metric depends on your problem and priorities.

Classification Metrics

The Confusion Matrix

For binary classification, we have four possible outcomes:

Predicted Positive Predicted Negative
Actual Positive True Positive (TP)
✓ Correct 		False Negative (FN)
✗ Miss
Actual Negative False Positive (FP)
✗ False Alarm 		True Negative (TN)
✓ Correct

Key Metrics

Accuracy

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Overall correctness. Good for balanced datasets, misleading for imbalanced data.

Precision

Precision = TP / (TP + FP)

Of predicted positives, how many are actually positive? Important when false positives are costly.

Recall (Sensitivity)

Recall = TP / (TP + FN)

Of actual positives, how many did we find? Important when missing positives is costly.

F1 Score

F1 = 2 × (Precision × Recall) / (Precision + Recall)

Harmonic mean of precision and recall. Balances both metrics.

Overfitting and Underfitting

⚠️ Overfitting

Model memorizes training data but fails on new data. High training accuracy, low test accuracy. Solution: More data, regularization, simpler model.

⚠️ Underfitting

Model is too simple to capture patterns. Low accuracy on both training and test data. Solution: More complex model, better features, longer training.
