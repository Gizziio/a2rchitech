# Final Project: Build Your Own AI Application

## Overview

For your final project, you will build an end-to-end AI/ML application that demonstrates 
the skills you've learned throughout this course. This is your opportunity to explore a 
topic of personal interest and create something meaningful.

## Project Options

Choose ONE of the following project tracks:

### Option 1: Image Classification System
Build a computer vision application that classifies images into categories.

**Ideas:**
- Plant disease detection from leaf images
- Handwritten digit/character recognition
- Object detection for a specific domain (tools, vehicles, etc.)
- Facial expression recognition
- Medical image analysis (X-ray, skin lesion classification)

**Requirements:**
- Collect or use an existing image dataset (minimum 1,000 images)
- Implement data augmentation
- Train a CNN (custom or transfer learning with ResNet/EfficientNet)
- Achieve at least 80% test accuracy
- Create a simple web interface or API for predictions

---

### Option 2: Natural Language Processing Application
Build an NLP system that processes and understands text.

**Ideas:**
- Sentiment analysis for product reviews or social media
- Text summarization tool
- Question-answering system on a specific domain
- Named entity recognition for a custom domain
- Content classification (news categorization, spam detection)

**Requirements:**
- Use a pre-trained transformer model (BERT, RoBERTa, or DistilBERT)
- Fine-tune on your specific task
- Evaluate using appropriate NLP metrics (F1, BLEU, ROUGE)
- Create a demo interface where users can input text

---

### Option 3: Predictive Analytics System
Build a system that makes predictions based on structured data.

**Ideas:**
- House price prediction
- Stock price trend prediction
- Customer churn prediction
- Demand forecasting for retail
- Credit risk assessment

**Requirements:**
- Comprehensive exploratory data analysis
- Feature engineering and selection
- Compare at least 3 different algorithms
- Implement proper cross-validation
- Deploy as a web service or create interactive visualizations

---

### Option 4: Creative/Generative AI Project
Build a generative AI application.

**Ideas:**
- Text generation tool (stories, poetry, code)
- Style transfer for images
- Music generation
- Chatbot for a specific domain

**Requirements:**
- Use appropriate generative models (GPT-2, StyleGAN, etc.)
- Fine-tune on domain-specific data if needed
- Implement controls for generation parameters
- Evaluate outputs qualitatively and/or quantitatively

---

## Project Deliverables

### 1. Code Repository (40%)
- Well-organized, documented code
- README with setup and usage instructions
- Requirements.txt or environment.yml
- Jupyter notebooks showing exploration and development process

### 2. Technical Report (30%)
5-10 page report including:
- **Introduction:** Problem statement and motivation
- **Data:** Source, collection process, preprocessing steps
- **Methodology:** Models and techniques used with justification
- **Results:** Performance metrics, visualizations, analysis
- **Discussion:** What worked, what didn't, lessons learned
- **Conclusion:** Summary and future improvements

### 3. Demo Video (15%)
3-5 minute video demonstrating:
- Project overview
- Live demonstration of the application
- Explanation of key technical decisions
- Reflection on challenges faced

### 4. Presentation (15%)
10-minute presentation to the class (recorded or live) covering:
- Problem and solution overview
- Technical approach
- Demo of working system
- Q&A

   - Model: Transfer learning with DenseNet121
   - Accuracy: 92% on test set

2. **Amazon Product Review Sentiment**
   - Dataset: 500K Amazon reviews
   - Model: Fine-tuned DistilBERT

   - Dataset: Hourly energy usage data
   - Models: LSTM, Prophet, XGBoost comparison

## Tips for Success

1. **Start early** - Data collection and cleaning often take longer than expected
2. **Keep it simple first** - Get a baseline working before adding complexity
3. **Document as you go** - Don't wait until the end to write your report
4. **Ask for help** - Use the Q&A forum and office hours
5. **Focus on learning** - The process matters more than perfect results

## Academic Integrity

- You may use existing code and tutorials as references, but cite them
- You may use AI assistants (ChatGPT, Copilot) for coding help, but understand and be able to explain all submitted code
- All writing (report, documentation) must be your own
- Collaboration on ideas is encouraged, but each project must be individual work
