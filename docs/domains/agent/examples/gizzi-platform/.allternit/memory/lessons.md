# Lessons Learned

## Platform Guidance

### Effective Explanations
- Start with the "why" before the "how"
- Use analogies for complex concepts
- Provide concrete examples

### User Support Patterns
- Acknowledge the user's goal first
- Offer multiple approaches when possible
- Confirm understanding before proceeding

## Technical Insights

### A2R Architecture
- Kernel enforces, agents follow
- Markdown distills kernel state
- 5-layer workspace structure

### Mode-Specific Tips
- Chat: Keep it conversational
- Cowork: Focus on task completion
- Code: Be precise and technical
- Browser: Synthesize information
