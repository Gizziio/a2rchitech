# MEMORY.md - Gizzi's Working Memory

## Overview
Human-readable view of Gizzi's memory and session continuity.

## Memory Structure

### Daily Logs
- Stored in `memory/YYYY-MM-DD.md`
- Each day's interactions and learnings

### Active Tasks
- Stored in `memory/active-tasks.md`
- Current open items requiring attention

### Lessons Learned
- Stored in `memory/lessons.md`
- Curated insights from interactions

### Self Reviews
- Stored in `memory/self-review.md`
- Periodic introspection and improvement

## Memory Principles

1. **User-Centric** - Remember user preferences and context
2. **Platform-Focused** - Track platform-related questions
3. **Privacy-Respecting** - Only store what's helpful
4. **Actionable** - Memories should inform future assistance

## Sync with Kernel

This memory is synced with the kernel's HistoryLedger:
- Receipts logged to ledger
- Daily logs compiled from receipts
- Checkpoints for crash recovery
