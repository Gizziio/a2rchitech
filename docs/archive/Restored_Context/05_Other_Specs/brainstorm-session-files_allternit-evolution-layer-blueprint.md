2. Iteration > Single-pass reasoning (InftyThink+, LLaDA 2.1)

	•	Learn when to summarize.

1️⃣ Memory Evolution Engine (MEE)

Purpose

Move from static MEMORY.md to executable, domain-adaptive memory systems.

Core Idea

Memory schema is code.
Schemas compete.
Better ones survive.

⸻

1.1 Architecture

/evolution/memory/
    archive/
    candidates/
    evaluator/
    schema_runtime/
    logs/

archive/

Stores prior memory schema implementations.

Each entry:

schema_id
domain
retrieval_strategy
update_strategy
performance_metrics

candidates/

New memory designs proposed by meta-agent.

evaluator/

Benchmarks:
	•	Retrieval precision
	•	Context compression ratio
	•	Impact on task success
	•	Token overhead

schema_runtime/

Executes active schema.

⸻

1.2 Operation Loop
	1.	Agent executes tasks.
	2.	Observability layer logs:
	•	Retrieval misses
	•	Redundant context
	•	Token bloat
	•	Recall failures
	3.	Meta-memory agent:
	•	Proposes alternative schema (graph, task signatures, risk schema, etc.)
	•	Runs shadow evaluation
	4.	If outperforming baseline → promote.

⸻

1.3 Key Constraint

Memory evolution cannot:
	•	Modify SOT
	•	Modify Contracts
	•	Alter historical truth

It can only change:
	•	Retrieval organization
	•	Indexing
	•	Summarization strategy
	•	Update heuristics
