Agentic Frontier Field Guide — v0.1

Living document. Sessions will be aggregated into this file as the doctrine, tactics, and patterns of AI‑native engineering evolve.

⸻

Core Thesis

Software is no longer written; it is specified and orchestrated.

Competitive advantage shifts from implementation speed to:
	•	Specification quality
	•	Loop orchestration
	•	Failure‑mode modeling
	•	Operator skill

Code becomes an intermediate artifact.

⸻

1. Labor Model Bifurcation

Developer (Commodity)
	•	Ticket → code translation
	•	Globally arbitraged
	•	Async
	•	Loop‑driven

Engineer (Scarce)
	•	System architecture
	•	Requirements synthesis
	•	Constraint modeling
	•	Security analysis
	•	Loop governance

Conclusion: Humans who only transform tickets into code are structurally obsolete.

⸻

2. Code Is No Longer the Moat

Weak moats:
	•	Source secrecy
	•	Licensing
	•	Proprietary repos

Strong moats:
	•	Domain mastery
	•	Distribution
	•	Regulatory positioning
	•	Data gravity
	•	Brand

Rule: If your moat is “we have the code,” you do not have a moat.

⸻

3. Tools Are Noise

AI coding products differ mostly in UX.

Durable advantages come from understanding:
	•	Task decomposition
	•	Deterministic loops
	•	Context isolation
	•	Spec‑first workflows

Focus on patterns, not brands.

⸻

4. Ralph‑Style Orchestration

Key insight: context reset per loop.
	•	Each iteration starts with a fresh window
	•	Memory persists only in specs
	•	Prevents degradation from competing goals

Analogy:
	•	Context = cache
	•	Specs = disk

⸻

5. Specifications Replace Source Code

Specs act as:
	•	Institutional memory
	•	Loop anchors
	•	Execution constraints

Properties:
	•	Generated through dialogue
	•	Continuously refined
	•	Cheap to rewrite

Effects:
	•	Prototypes ≈ free
	•	Refactors ≈ cheap
	•	Rewrites ≈ routine

The bottleneck is clarity.

⸻

6. Hiring Signal Inversion

New baseline:
	•	Explain how to design a coding agent
	•	Show orchestration patterns

Risks:
	•	Juniors without AI fluency
	•	Seniors refusing adaptation

Velocity divergence becomes career‑limiting.

⸻

7. AI Amplifies Skill

AI multiplies operator quality.

Strong engineers:
	•	Become force multipliers

Weak engineers:
	•	Become faster liabilities

Edge comes from:
	•	Closing feedback loops
	•	Encoding judgment into specs
	•	Automation discovery

⸻

8. Open Source Re‑Evaluation

Automation friction:
	•	Human maintainers
	•	PR queues
	•	Governance

Strategy:
	•	Generate commodity code
	•	Depend only where verification is possible

Hard exceptions:
	•	Cryptography
	•	Security primitives

⸻

9. Language Tiering for Agents

S‑Tier properties:
	•	Source‑searchable
	•	Strong typing
	•	Deterministic failure

Examples:
	•	Rust
	•	TypeScript
	•	Python + Pydantic

F‑Tier properties:
	•	Binary‑first ecosystems
	•	Opaque dependency graphs

Examples:
	•	Java
	•	.NET

Tradeoff: Rust is costly when generations are wrong.

⸻

10. Corporate Failure Mode

Large organizations:
	•	Move slowly
	•	Over‑govern
	•	Transform over years

Small teams:
	•	Ralph systems
	•	Ship relentlessly
	•	Undercut pricing

Innovator’s dilemma, compressed.

11. Post‑Human‑Centric Computing

Past optimized for humans:
	•	TTYs
	•	ENV vars
	•	Backwards compatibility

Future optimized for agents:
	•	Machine‑first interfaces
	•	Minimal control planes
	•	Deterministic orchestration surfaces

We have a new computer.
