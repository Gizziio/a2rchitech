# Wireframe to Implementation Mapping

This document maps the conceptual wireframe sections to actual components in the implementation for the marketplace feature.

## 1. Overall Layout

### Wireframe Concept
- Header with title and description
- Filter controls (search, categories, sort options)
- Asset grid/list view
- Asset detail panel/modal
- Import button functionality

### Implementation Components
- `MarketplaceView.tsx`: Main container component that orchestrates the layout
- `TwoModeRegistryView.tsx`: Wrapper component that provides the toggle between Registry and Marketplace modes

## 2. Header Section

### Wireframe Elements
- Title: "Marketplace"
- Subtitle: Description of the marketplace functionality
- Stats: Total assets count, imported assets count

### Implementation
- Located in `MarketplaceView.tsx` within the `.marketplace-header` div
- Contains:
  - Header title (`<h1>Marketplace</h1>`)
  - Description paragraph
  - Stats display showing total and imported asset counts

## 3. Filter Controls Section

### Wireframe Elements
- Search input field
- Category filter buttons (All, Capsules, Agents, Packs)
- Sort options (Popular, Newest, Top Rated, Name)

### Implementation
- Located in `MarketplaceView.tsx` within the `.marketplace-filters` div
- Search input: Controlled by `filters.search` state
- Category filters: Implemented as toggle buttons controlling `filters.type` state
- Sort options: Implemented as buttons controlling `filters.sortBy` state

## 4. Asset Grid/List Section

### Wireframe Elements
- Grid of asset cards
- Each card contains:
  - Icon and name
  - Version number
  - Trust badge
  - Rating stars
  - Download count
  - Author information
  - Description
  - Tags
  - Action buttons (Import, Details)

### Implementation
- Located in `MarketplaceView.tsx` within the `.assets-container` div
- Each asset card is rendered as a `.asset-card` div
- Cards are generated by mapping over the `filteredAssets` array
- Individual card elements include:
  - Asset icon from `getAssetTypeIcon()` function
  - Name and version display
  - Trust tier badge with color coding
  - Star rating display with `getStarRating()` helper
  - Stats (downloads, author)
  - Description text
  - Tag chips
  - Action buttons (Import/Details)

## 5. Asset Detail Modal

### Wireframe Elements
- Modal overlay
- Asset icon and name
- Version information
- Detailed description
- Metadata (author, downloads, rating)
- Action button (Import)

### Implementation
- Located in `MarketplaceView.tsx` within the conditional `.asset-modal` div
- Triggered by clicking on an asset card
- Displays detailed information from the selected asset
- Includes Import button that calls `handleImport()` function

## 6. Two-Mode Toggle

### Wireframe Concept
- UI switch to toggle between Registry and Marketplace views

### Implementation
- `TwoModeRegistryView.tsx`: New component that provides the toggle functionality
- Contains a button group with "Registered" and "Marketplace" options
- Renders either `<RegistryView />` or `<MarketplaceView />` based on the selected mode
- Integrated into the main app flow via `App.tsx`

## 7. Data Flow

### Wireframe Concept
- Search/filter inputs affect the displayed assets
- Clicking an asset shows its details
- Import action adds the asset to the local registry

### Implementation
- `marketplaceClient.ts`: Boundary module that handles all marketplace data operations
- `searchAssets()` method makes actual API calls to `/api/v1/marketplace/search` when filters change
- `getAssetDetail()` method makes actual API calls to `/api/v1/marketplace/assets/{id}` for detailed view (though not currently used in the UI)
- `importAsset()` method makes actual API calls to `/api/v1/marketplace/import/{id}` triggered by Import buttons
- State management in `MarketplaceView.tsx` coordinates the UI updates

## 8. Styling and Visual Design

### Wireframe Concept
- Clean, modern interface
- Consistent color scheme
- Responsive layout

### Implementation
- CSS classes defined in the component's stylesheet
- Color scheme implemented via inline styles and CSS variables
- Responsive design achieved through flexbox and media queries
- Animation effects using CSS transitions and keyframes

## 9. Loading and Empty States

### Wireframe Concept
- Loading spinner during data fetch
- Empty state when no assets match filters

### Implementation
- Loading state managed by `loading` boolean in `MarketplaceView.tsx`
- Loading spinner displayed when `loading` is true
- Empty state message displayed when `filteredAssets.length` is 0
