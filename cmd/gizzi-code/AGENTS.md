# gizzi-code Agent Guide

## Database

- **Schema**: Drizzle schema lives in `src/**/*.sql.ts`.
- **Naming**: tables and columns use snake_case; join columns are `<entity>_id`; indexes are `<table>_<column>_idx`.
- **Migrations**: generated by Drizzle Kit using `drizzle.config.ts` (schema: `./src/**/*.sql.ts`, output: `./migration`).
- **Command**: `bun run db generate --name <slug>`.
- **Output**: creates `migration/<timestamp>_<slug>/migration.sql` and `snapshot.json`.
- **Tests**: migration tests should read the per-folder layout (no `_journal.json`).

## Vault Subsystem

The knowledge vault is an Obsidian-compatible Markdown store with YAML frontmatter, `[[wikilinks]]`, and a SQLite FTS5 search index.

### Key modules

| Path | Purpose |
|------|---------|
| `src/vault/index.ts` | `VaultManager` — initialize, CRUD notes, query |
| `src/vault/io.ts` | `ensureVaultStructure`, `sanitizeFilename`, read/write |
| `src/vault/search.ts` | `VaultIndex` — SQLite FTS5 + entity/link/tag tables |
| `src/vault/graph.ts` | `EntityGraph` — backlinks, entity extraction |
| `src/vault/settings.ts` | User settings (connector enablement, OAuth creds, API keys) |
| `src/vault/connector.ts` | `VaultConnector` interface, registry, orchestrator |
| `src/vault/sync/index.ts` | `runSync()`, `runAllSyncs()` — entry points for sync |

### Connectors

All connectors implement `VaultConnector`:

```typescript
interface VaultConnector {
  id: string
  name: string
  authType: "oauth" | "api_key" | "none"
  init(): Promise<void>
  connect(): Promise<void>
  sync(vault: VaultManager): Promise<SyncResult>
  status(): Promise<ConnectorStatus>
}
```

Registered connectors:
- `gmail` — Google OAuth, fetches threads via Gmail API v1
- `calendar` — Google OAuth (shares token with Gmail)
- `fireflies` — API key auth, GraphQL transcripts

Connectors default to `enabled: false`. Users opt-in via `gizzi vault config`.

### CLI

```bash
gizzi vault init                    # Create vault structure
gizzi vault query <text>            # FTS5 search
gizzi vault write <folder> <title>  # Create note
gizzi vault sync [gmail|calendar|fireflies|all]
gizzi vault daily                   # Open today's note
gizzi vault graph                   # Rebuild entity graph
gizzi vault live                    # Update live notes
gizzi vault status                  # Stats
gizzi vault config                  # Interactive settings
```

## Cron / Daemon

The daemon (`gizzi daemon`) runs a background cron scheduler on port **3031** with SQLite persistence.

### Architecture

```
Daemon (src/daemon/main.ts)
  └── CronDaemon (src/runtime/automation/cron/daemon.ts)
        └── CronService (src/runtime/automation/cron/service.ts)
              ├── SQLite persistence (cron.db)
              ├── Job scheduler (cron + interval)
              └── Job executors
```

### Job types

| Type | Executor | Use case |
|------|----------|----------|
| `shell` | `Bun.spawn` | Shell commands |
| `http` | `fetch` | Webhooks |
| `agent` | `AgentExecutor` | AI agent tasks |
| `cowork` | `CoworkExecutor` | Docker/VM/local runtime |
| `function` | `FunctionRegistry` | Pre-registered in-process functions |
| `vault` | `VaultSyncExecutor` | Vault sync (Gmail, Calendar, Fireflies, live notes) |

### Important: `function` jobs do NOT support `@/` paths in production

Dynamic `import("@/module")` cannot be resolved in a `bun build --compile` binary because the bundler cannot analyze runtime strings. All internal functions must be registered statically:

```typescript
import { registerFunction } from "@/runtime/automation/cron/executors/function-registry"
import { runSync } from "@/vault/sync"

registerFunction("vault-sync", runSync)
```

Then reference by name in job config:

```typescript
CronService.create({
  type: "function",
  config: { function: "vault-sync", args: ["gmail"] },
})
```

### Vault sync job registration

`registerDefaultVaultJobs()` (called at daemon startup) reconciles cron jobs with connector settings:

- Creates a `vault` job for each **enabled** connector
- Updates schedule if the cron expression changed
- Deletes jobs for connectors that are **disabled**
- Creates/removes live-notes job based on `settings.liveNotes.enabled`
- Uses promise-locking to prevent duplicate registration races

Connector schedules:
- `gmail`: `0 2 * * *` (daily 2:00 AM)
- `calendar`: `30 2 * * *` (daily 2:30 AM)
- `fireflies`: `0 3 * * *` (daily 3:00 AM)
- `live-notes`: `0 */6 * * *` (every 6 hours)

### Entry points

```bash
gizzi daemon start      # Start daemon (foreground)
gizzi daemon start --background
gizzi daemon stop       # Stop daemon
gizzi daemon status     # Show status
```

## Brain Memory

Triple-store memory system (episodic / semantic / procedural) with entity graph and BRAIN.md sync.

| Path | Purpose |
|------|---------|
| `src/runtime/brain/brain.service.ts` | `BrainService` — CRUD chunks, entities, relations, BRAIN.md sync |
| `src/runtime/brain/memory.sql.ts` | Drizzle schema for `brain_memory_chunk`, `brain_memory_embedding`, `brain_memory_entity`, `brain_memory_relation` |
| `src/cli/commands/brain.ts` | CLI: `status`, `remember`, `recall`, `entities`, `sync` |

### Retrieval strategy (no embeddings required)
1. Recency + importance ranking
2. Keyword `LIKE` search on content
3. Access frequency boost

With `GIZZI_EMBEDDING_PROVIDER` set (openai/ollama), semantic cosine-similarity search is used.

### Migration

## Voice

Local-first voice interface using open-source tools.

| Path | Purpose |
|------|---------|
| `src/runtime/voice/voice.ts` | `listen()`, `speak()`, `transcribe()`, `voiceChat()` |
| `src/cli/commands/voice.ts` | CLI: `listen`, `speak`, `transcribe`, `chat` |

### Dependencies
- **STT**: `openai-whisper` (brew install whisper)
- **TTS**: macOS `say` (Linux: `espeak` / `spd-say`)
- **Recording**: `sox` or `ffmpeg`

Voice chat currently echoes transcriptions. Wire to AI runtime via `getResponse` callback.

## VM Runtime

| Path | Purpose |
|------|---------|
| `src/runtime/vm/vfkit-manager.ts` | `VFKitManager` — start/stop/status/exec via vfkit process |

| `src/runtime/vm/index.ts` | Public exports |
| `src/cli/commands/vm.ts` | CLI: `start`, `stop`, `restart`, `status`, `setup`, `exec` |

### Setup
```bash
brew install vfkit
bun run vm:download   # CI-built images
```

Architecture: Host vfkit process → Apple VZ VM → VSOCK Unix socket → Guest agent (`allternit-vm-executor`).
