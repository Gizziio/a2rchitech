A Real Adapter Implementation

Let's examine the AiSdkReplyAdapter from @allternit/provider-adapters — the same adapter running in production that normalizes streams from OpenAI, Anthropic, Gemini, and local models into a single canonical event format.

packages/@allternit/provider-adapters/src/ai-sdk.tsTypeScript
/**
 * AiSdkReplyAdapter
 *
 * Converts AI SDK v6 TextStreamPart events into canonical ReplyEvents.
 * Works for ALL AI SDK providers with zero provider-specific code.
 */
export class AiSdkReplyAdapter {
  private readonly openedTextIds = new Set<string>();
  private readonly pendingTools = new Map<string, { toolName: string }>();
  private hasNativeReasoning = false;

  process(part: AiSdkStreamPart): void {
    switch (part.type) {
      case "text-delta": {
        if (!this.openedTextIds.has(part.id)) {
          this.openedTextIds.add(part.id);
          this.emit({
            type: "reply.item.added",
            itemId: part.id,
            kind: "text",
          });
        }
        this.emit({
          type: "reply.text.delta",
          itemId: part.id,
          delta: part.text,
        });
        break;
      }

      case "tool-call": {
        const { toolCallId, toolName, args } = part;
        this.emit({
          type: "tool_call.started",
          toolCallId,
          toolName,
          input: args,
        });
        break;
      }

      case "tool-result": {
        const itemId = this.toolCallIdToItemId.get(part.toolCallId)
          ?? part.toolCallId;
        this.emit({
          type: "tool_call.completed",
          itemId,
          toolCallId: part.toolCallId,
          output: part.result,
        });
        break;
      }

      case "error": {
        this.emit({
          type: "reply.failed",
          error: part.error instanceof Error
            ? part.error.message
            : "stream error",
        });
        break;
      }
    }
  }

  async consume(stream: AsyncIterable<AiSdkStreamPart>): Promise<void> {
    try {
      for await (const part of stream) {
        this.process(part);
      }
    } catch (err) {
      this.emit({
        type: "reply.failed",
        error: err instanceof Error ? err.message : "stream consume error",
      });
    }
  }
}

What makes this production-grade? The adapter handles every edge case: providers that skip text-start, reasoning blocks embedded in plain text (<thinking>), MCP app results, tool call deduplication, and graceful degradation on stream errors. One adapter, all providers.

surfaces/allternit-platform/src/lib/error-rate-limiter.tsTypeScript
export interface RateLimitConfig {
  maxAttempts?: number;
  windowMs?: number;
  cooldownMs?: number;
}

const DEFAULT_CONFIG: Required<RateLimitConfig> = {
  maxAttempts: 3,
  windowMs: 60000,      // 1 minute
  cooldownMs: 300000,    // 5 minutes
};

class ErrorRateLimiter {
  private errors = new Map<string, ErrorEntry>();

  shouldShow(errorKey: string): boolean {
    const now = Date.now();
    const entry = this.errors.get(errorKey);

    if (!entry) {
      this.errors.set(errorKey, {
        count: 1,
        firstSeen: now,
        lastSeen: now,
        suppressed: false,
      });
      return true;
    }

    // In cooldown?
    if (entry.suppressed) {
      const elapsed = now - entry.lastSeen;
      if (elapsed < this.config.cooldownMs) {
        entry.lastSeen = now;
        return false;
      }
      entry.suppressed = false;
      entry.count = 1;
      return true;
    }

    // Window expired?
    if (now - entry.firstSeen > this.config.windowMs) {
      entry.count = 1;
      entry.firstSeen = now;
      return true;
    }

    entry.count++;
    if (entry.count > this.config.maxAttempts) {
      entry.suppressed = true;
      return false;
    }
    return true;
  }

  getSummary(): { totalErrors: number; suppressedErrors: number; activeErrors: number } {
    let suppressed = 0, active = 0;
    for (const entry of this.errors.values()) {
      entry.suppressed ? suppressed++ : active++;
    }
    return { totalErrors: this.errors.size, suppressedErrors: suppressed, activeErrors: active };
  }
}

The ErrorRateLimiter prevents error notification spam. But more importantly, the same pattern powers API rate limiting: track keys per provider, enforce windows, and suppress (or queue) requests that exceed the threshold.

surfaces/allternit-platform/src/hooks/use-retry.tsTypeScript
export interface RetryConfig {
  maxAttempts?: number;
  initialDelay?: number;
  maxDelay?: number;
  backoffMultiplier?: number;
}

const DEFAULT_CONFIG: Required<RetryConfig> = {
  maxAttempts: 3,
  initialDelay: 1000,
  maxDelay: 30000,
  backoffMultiplier: 2,
};

export async function withRetry<T>(
  operation: () => Promise<T>,
  config: RetryConfig = {}
): Promise<T> {
  const finalConfig = { ...DEFAULT_CONFIG, ...config };

  for (let attempt = 1; attempt <= finalConfig.maxAttempts; attempt++) {
    try {
      return await operation();
    } catch (error) {
      const err = error instanceof Error ? error : new Error(String(error));

      if (attempt >= finalConfig.maxAttempts) {
        throw err;
      }

      const delay = Math.min(
        finalConfig.initialDelay * Math.pow(finalConfig.backoffMultiplier, attempt - 1),
        finalConfig.maxDelay
      );

      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }

  throw new Error("Retry loop exited unexpectedly");
}

Exponential backoff with jitter (implied by the backoffMultiplier) prevents thundering herds when a provider recovers. The maxDelay cap ensures users never wait more than 30 seconds between attempts.

    model: String,
    client: Client,

#[async_trait]
impl LLMProvider for OpenAIProvider {
    async fn complete(&self, system: &str, user: &str) -> Result<String> {
        let request = CompletionRequest {
            model: self.model.clone(),
            messages: vec![...],
            temperature: 0.0,
        };

        let res = self
            .client
            .post("https://api.openai.com/v1/chat/completions")
            .header("Authorization", format!("Bearer {}", self.api_key))
            .json(&request)
            .send()
            .await
            .context("Failed to send request to OpenAI")?;

        if !res.status().is_success() {
            let error_text = res.text().await?;
            anyhow::bail!("OpenAI API error: {}", error_text);
        }

        let body: CompletionResponse = res
            .json()
            .await
            .context("Failed to parse OpenAI response")?;

        body.choices
            .first()
            .map(|c| c.message.content.clone())
            .ok_or_else(|| anyhow::anyhow!("No choices in response"))
    }
}

Resilience

Failover in Practice

What happens when the primary runtime backend goes offline? Allternit's gateway proxy implements automatic failover with health-checked candidate rotation.

The Gateway Proxy Failover

surfaces/allternit-platform/src/lib/runtime-gateway-proxy.tsTypeScript
export async function proxyGatewayRequest(
  request: NextRequest,
  upstreamPath: string,
  options?: {
    allowStatusesUnder500?: boolean;
    retryOnStatuses?: number[];
  },
): Promise<Response> {
  const candidates = await getGatewayCandidates();
  const allowStatusesUnder500 = options?.allowStatusesUnder500 ?? true;
  const retryOnStatuses = new Set(options?.retryOnStatuses ?? []);

  for (const candidate of candidates) {
    const targetUrl = `${candidate.base}${upstreamPath}`;
    try {
      const upstream = await fetch(targetUrl, {
        method,
        headers: buildHeaders(request, candidate, options),
        body: bodyBuffer?.byteLength > 0 ? bodyBuffer : undefined,
        signal: AbortSignal.timeout(15_000),
        duplex: bodyBuffer ? "half" : undefined,
      });

      if (retryOnStatuses.has(upstream.status)) {
        continue; // Try next candidate
      }

      if (upstream.ok || (allowStatusesUnder500 && upstream.status < 500)) {
        return buildResponse(upstream);
      }
    } catch {
      // Try the next candidate.
    }
  }

  return NextResponse.json(
    { error: "Runtime backend is unavailable" },
    { status: 503 },
  );
}

Backend Snapshot Caching

To avoid resolving the backend on every request, the client caches a snapshot with a 15-second TTL:

surfaces/allternit-platform/src/lib/runtime-backend-client.tsTypeScript
const SNAPSHOT_TTL_MS = 15_000;

export async function loadRuntimeBackendSnapshot(
  options: { force?: boolean } = {},
): Promise<ClientRuntimeBackendSnapshot> {
  const cached = getRuntimeBackendSnapshotSync();
  const fetchedAt = Date.parse(cached.fetched_at);

  if (
    !options.force &&
    Number.isFinite(fetchedAt) &&
    Date.now() - fetchedAt < SNAPSHOT_TTL_MS
  ) {
    return cached; // Fresh enough
  }

  if (inflightSnapshotPromise && !options.force) {
    return inflightSnapshotPromise; // Deduplicate
  }

  inflightSnapshotPromise = fetch("/api/v1/runtime/backend", { ... })
    .then(async (response) => {
      const payload = await response.json();
      return applyRuntimeBackendSnapshot(payload);
    })
    .catch(() => cached)
    .finally(() => {
      inflightSnapshotPromise = null;
    });

  return inflightSnapshotPromise;
}
⚠
Design Note: The snapshot uses request deduplication (inflightSnapshotPromise) so that 50 concurrent requests don't trigger 50 backend resolves. Only the first request fetches; the rest await the same promise.

Gateway Client Reconnection Telemetry

The OpenClaw gateway client implements exponential backoff reconnection with capped jitter:

views/openclaw/gateway-client.tsTypeScript
private backoffMs = 800;

private scheduleReconnect(): void {
  if (this.closed) return;

  const timeoutMs = this.backoffMs;
  this.backoffMs = Math.min(Math.round(this.backoffMs * 1.7), 15_000);

  if (this.reconnectHandle != null) {
    window.clearTimeout(this.reconnectHandle);
  }

  this.reconnectHandle = window.setTimeout(() => {
    this.reconnectHandle = null;
    this.connect();
  }, timeoutMs);
}

Notice the multiplier (1.7x) and cap (15s). On successful connection, backoffMs resets to 800ms. This is jittered exponential backoff in practice.

Multi-Provider AI Gateway

Create an adapter that routes chat requests to OpenAI, Anthropic, or a local Ollama model based on availability, cost, and user preference. This is exactly what Allternit's production gateway does.

Requirements

  1. 1. Gateway Class: Implement a MultiProviderGateway that maintains a priority-ordered list of providers (OpenAI → Anthropic → Ollama).
  2. 2. Health Checks: Each provider must implement a healthCheck() method that returns { healthy: boolean; latencyMs: number } within 3 seconds.
  3. 3. Cost Tracking: Track estimated cost per request using token counts and per-provider pricing tables. Expose a getCostReport() method.
  4. 4. Response Normalization: All providers must return a unified GatewayResponse with { text: string; model: string; usage: { promptTokens, completionTokens } }.
  5. 5. Fallback Chain: If the primary provider fails or is unhealthy, automatically try the next. If all fail, return a structured error.
  6. 6. Tests: Write tests that simulate provider outages (using nock or mock providers) and verify the fallback chain works correctly.
